# AGENTS.md

## 项目开发规则

- 修改代码前先理解当前结构和已有约定。
- 保持变更范围清晰，避免大范围重构无关代码。
- 每个 Step 都需要有清晰的验收结果。

## AI 执行约束

- 不要提交密钥、令牌、证书或其他敏感信息。
- 不要读取或打包 .env、密钥文件、node_modules、dist、target 等敏感或生成目录。
- 不要在未明确要求时自动修改业务代码。
- 对外部工具、执行结果和失败原因保持可追溯记录。

## Git 工作区约定

- 主目录 `agentflow` 用于保持主分支和远端同步。
- Codex 后续开发优先使用独立 worktree，例如 `agentflow-codex`。
- 不要把同一个分支同时检出到多个 worktree；每个任务使用独立分支。
- 提交仍然提交到同一个 Git 仓库和远端，只是从当前工作的 worktree 执行。
