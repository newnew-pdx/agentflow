# AgentFlow 愿景

AgentFlow 希望把依赖人工协调的 AI 辅助开发过程，逐步转变为透明、本地优先且可恢复的工程工作流。

目标闭环是：

`规划 -> 审批 -> 执行 -> 验证 -> 审查 -> 修复或重新规划 -> 提交 -> 更新记忆 -> 下一步`

当前开发者往往需要在网页 AI、代码代理、测试命令和 Git 操作之间手动搬运信息。计划可能停留在聊天记录中，执行结果与审查意见缺少统一结构，下一轮工作容易丢失关键上下文。AgentFlow 希望以本地 CLI 为入口，用明确协议描述各阶段输入与输出，并保存必要的项目状态和记忆。

## 设计原则

- **本地优先**：核心状态和协议产物保存在开发者可检查的位置。
- **过程透明**：规划、执行、验证和审查具有明确边界。
- **开发者审批**：高影响操作保留显式审批点。
- **可恢复**：流程中断后，可以根据本地状态了解进度并继续工作。
- **渐进实现**：先稳定骨架和协议，再逐步接入执行器、数据库与外部代理。

Step0 只负责 CLI 与目录骨架。完整闭环是后续目标，并非当前已经具备的能力。
